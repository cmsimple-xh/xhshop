XH-Shop

A small simple shop for CMSimple_XH

Requirements

  • CMSimple_XH ≥ 1.7.0
  • PHP ≥ 5.3.0
  • Plugin FA_XH (is already included since CMSimple_XH 1.7)
  • Plugin FancyBox recommended – but the shop also works without it.
    (the link is a direct download)

Installation

  1. Backup your CMSimple_XH installation from your server (backup copy).
  2. Unpack the downloaded ZIP archive on your computer.

After unzipping you will find 3 folders: content, plugins and userfiles. These folders represent the target folders of your CMSimple_XH installation – that means the contents of these folders have to be copied to the corresponding folders of your CMSimple_XH installation.

The folder content
Copy the folder xhshop completely into the folder content of your CMSimple_XH installation. This folder contains the catalog.php with all data of all shop items and categories that will be delivered for demonstration. The storage in the content folder has the advantage that the data is preserved in case of a new installation or an update of the shop. In addition, this folder contains another one: tmp_orders, which is used to temporarily store temporary data for PayPal orders.

The folder plugins
Copy the folder xhshop completely into the plugins folder of your CMSimple_XH installation. This folder contains the XH-Shop plugin. Make sure you have write permissions (0755) for the subdirectories config/, css/ and languages/ here.

The folderuserfiles
Copy the folder images/shop/ contained therein completely into the folder userfiles/images/ of your CMSimple_XH installation. This folder contains sample images that are delivered for demonstration purposes. You can remove these demo images later, once you have created your own articles with your own images.

Administration

Settings

Admin-Menu: Plugins ⇒ Xhshop ⇒ Config.

Here you can change the existing sample settings of the shop. When you move the mouse over the help icons (?), hints about the settings are displayed.

Language

Admin-Menu: Plugins ⇒ Xhshop ⇒ Language.

All texts that are output by the plugin in the frontend or in the backend can be found here and can be changed – depending on the currently used language.

Some special link addresses (e.g. the shop main page, the terms and conditions and the shipping costs) are set here (right above in the Config section). Also the settings for decimal point/comma and thousands separator are done here for the whole shop.

Stylesheet

Admin-Menu: Plugins ⇒ Xhshop ⇒ Stylesheet.

The CSS styles of the XH-Shop can be customized here.

Open a Shop

Log in to CMSimple_XH and create a new page Shop with the page manager (admin menu ⇒ Pages). On this page, you enter the plugin call. The "normal" call for the XH shop is:

{{{display_shop();}}}

Note: If you copy the code from here, then you have to select "Paste as Text" in the editor before pasting. At the end, the call in the html view of the editor should look like this:

<div>{{{display_shop();}}}</div>

It is easily possible to place other elements on the shop page above or below this call.

Shop-Templates

The HTML output of the XH-Shop is completely controlled by templates. These can be found in the plugin folder /templates.

If you prefer not to change the original files – and you should really only do this with extreme caution – then simply copy the files you want (for the frontend, for example) into the xhshop/theme/frontend/ folder. You can safely experiment with these files. XH-Shop will look at every call if there are files in this folder. If so, it will use these files as priority for the output. If something went wrong, you can rename or remove your changed files – XH-Shop will then use the original files again.
